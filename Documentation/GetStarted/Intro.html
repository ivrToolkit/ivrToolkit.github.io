Introduction

The ivrToolkit allows you to create Interactive Voice Response applications using .net 6.0 and up.

The ivrToolkit is designed to make auto dialers. It also allows you to make ivr call-ins to a server as well. For example, say a person needs to call in and log and absence. It uses a ScriptManager class which executes implementations of the AbstractScript class. I really need to document this process :)

History

There are currently There plugins(have a look at requirements for more detail):

  • ivrToolkit.Plugin.SipSorcery (beta) - Uses the SipSorcery c# project you can find here
  • ivrToolkit.Plugin.Dialogic.SIP - Uses Dialogic HMP 3.0 driver.
  • ivrToolkit.Plugin.Dialogic.Analog - Uses Dialogic SR 6.0 driver. and DM3/JCT boards.

Combinations of hardware and software that I have tried:

  • With ivrToolkit.Plugin.Sipcorcery I have tried Voip.MS with FreePBX.
  • VMWare with Windows Server 2019, Dialogic HMP 3.0 update 525
  • Windows Server 2016, Dialogic SR 6.0 update 271, Dialogic D/41H jct-ls, D/120 jct-ls
