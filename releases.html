ivrToolkit

An IVR toolkit library for dotNet developers wanting to write telephony applications. Plugins include a Dialogic driver and a windows sound card driver.

Release Source

release-1.1.0-alpha1.zip
  • Breaking change to IVoice interface. GetLine method signature is now:
         GetLine(int lineNumber, object data = null);
    Where data can be a string representing the device name of the Dialogic line. For Example: ':P_pdk_na_an_io:V_dxxxB1C1'
    This is only needed if your aren't able to define what you need in the voice.properties file. voice.deviceNamePattern.

  • New keyValue in voice.properties called 'voice.deviceNamePattern' which is defaulted to ':P_pdk_na_an_io:V_dxxxB{board}C{channel}'. This is accessed by the dialogic plugin using Voice.Current.DeviceNamePattern. Valid keywords are {line}, {board} and {channel}. Board and channel are calculated based on the line number. 4 lines per board.

  • Replaced dx_open with gc_OpenEx. This is supposed to allow SIP use with the Dialogic® PowerMedia™ HMP for Windows software. I'll be testing that next week.
release-1.0.zip
  • Added void RecordToFile(string filename, int timeoutMillisconds) to ILine interface. Modified the dialogic and simulator plugin to use the new method. The previous RecordToFile method would record indefinitely if the caller hung up during the record.
  • Fixed the GetLine method in Dialogic.cs so that it now properly caluclates the board and channel number for a line number greater than 4.
release-0.9.6.zip
  • Add volume control methods
  • Removed prompt terminator character 'T'. You can get the same result by using terminator 't' and specifying AllowBlank=false
release-0.9.5.zip
  • Fixed bug where dialing a number like '9,1112223333' would not dial the 9. This only happened when dial.customOutbound.enabled=true in voice.properties
  • Fixed bug where line status didn't switch from 'AcceptingCalls' to 'Connected' after the WaitRings method answered a call.
release-0.9.4.zip
  • Change the licensing from GPL to LGPL.
  • Modified prompt.attempts from 2 to 5 in voice.properties. However, I think this will be changed to prompt.noResponseAttempts instead since the intent was to stop endless looping of a prompt if the user hung up and the system did not detect the hangup.
